Prerequisites

Before using AI Conversation Analyzer, make sure you have the following.

Tip

AI Conversation Analyzer runs entirely in the browser. No server-side installation is required.

  • A modern browser (Chrome, Edge, Firefox, or Safari)
  • A local HTTP server (the app uses ES modules and cannot run from file://)
  • A session JSON file in OpenCode format, or a running OpenCode instance

Installation

Clone the repository and serve it locally.

Terminal 
git clone https://github.com/Conversation-analyzer/ai_conversation_analyzer.git
cd ai_conversation_analyzer

Run Locally

Start a local HTTP server using any of these methods.

Terminal 
python -m http.server 8080
Terminal 
npx serve .
Terminal 
php -S localhost:8080

Then open http://localhost:8080 in your browser.

App Settings

Configure the application defaults in config/analyzer.json.

Key Type Default Description
theme string "dark" Default theme ("dark" or "light"). Overridden by browser preference and localStorage.
defaultTab string "overview" Tab shown on startup. Options: "overview", "timeline", "messages", "tools", "insights".

Backend Config

Configure connection details in config/opencode.json.

Key Type Default Description
type string "opencode" Connection type identifier.
label string "OpenCode (local)" Display name shown in the connector picker.
protocol string "http" HTTP or HTTPS.
host string "127.0.0.1" Backend hostname or IP.
port number 4472 Backend port number.
timeout number 10000 Request timeout in milliseconds.

Adding Backends

Extend the analyzer to support new AI platforms by adding 3 files and a config entry.

  1. 1
    Create the connection class

    Add js/connections/myplatform.js implementing the get(path) method.

  2. 2
    Create the datasource class

    Add js/datasource/myplatform.js implementing getMessages(id) and getLatestSession().

  3. 3
    Create the parser

    Add js/parser/myplatform_parser.js to convert raw data into a ConversationSpec.

  4. 4
    Register in config

    Add your config to config/myplatform.json and list it in config/manifest.json.

Tip

Use the existing OpenCode implementation as a reference. The connection, datasource, and parser files are under 100 lines each.

The analyzer follows a unidirectional data flow pattern.

        |
        v

        v

File Structure

Key files and their responsibilities.

File Responsibility
js/app.js Application core. Manages state, event handlers, and orchestrates the render pipeline.
js/renderer.js All HTML rendering for Overview, Timeline, Messages, Tools, and Insights tabs.
js/grids.js AG Grid integration for sortable, filterable message and tool tables.
js/charts.js Plotly chart rendering — 7 chart types for the Insights tab.
js/extractor.js Six extraction functions that pull specific slices from ConversationSpec.
js/parser/opencode_parser.js Converts raw OpenCode JSON into a normalized ConversationSpec.
js/connections/ HTTP client classes for each supported backend.
js/datasource/ Data fetchers that use connections to retrieve sessions and messages.
config/ JSON configuration files for the app and each backend.

ConversationSpec

The normalized data model that all renderers consume.

Schema 
{
  version: "1.0",
  source: "opencode",             // "opencode" | "cursor" | "claude" | ...
  session: {
    id: "ses_...",
    title: "...",
    startedAt: "...",
    completedAt: "..."
  },
  assistant: {
    agent: "...",
    model: "...",
    provider: "..."
  },
  metrics: {
    inputTokens: 0,
    outputTokens: 0,
    reasoningTokens: 0,
    totalTokens: 0,
    cacheRead: 0,
    cacheWrite: 0,
    cost: 0
  },
  messages: [...],                // Compiled messages with parts
  timeline: [...],                // Sorted events (MESSAGE, REASONING, TOOL_START, TOOL_END)
  tools: [...],                   // Tool calls with duration and status
  metadata: {}
}

Extractors

Extract specific data slices from a ConversationSpec.

Function Returns
extractSession(conv) Session info, assistant details, and summary stats.
extractMetrics(conv) Token counts, cache stats, cost, and health score.
extractMessages(conv) Compiled messages grouped into conversation turns.
extractTools(conv) All tool calls with input, output, status, and duration.
extractTimeline(conv) Sorted timeline events for the event stream view.
extractSummary(conv) Quick summary stats (total messages, tools used, duration).

FAQ

Frequently asked questions.

What file formats are supported?

Currently, the analyzer supports OpenCode session JSON format. Support for Cursor, Claude Code, and generic JSON is planned on the roadmap.

Which browsers are supported?

Chrome, Edge, Firefox, and Safari. The app uses ES modules and IntersectionObserver, which are supported in all modern browsers.

Is there a CLI version?

No. AI Conversation Analyzer is a browser-based tool. There is no server-side component or CLI.

How is the health score calculated?

The health score (0-100) is computed from two factors: tool error rate and reasoning share. A high error rate or very low reasoning activity lowers the score.

Can I change the live polling interval?

Currently, the polling interval is hardcoded at 4 seconds. Configurable polling is planned for a future release.

Does my data leave my machine?

No. The analyzer runs entirely in your browser. When using file upload, data never leaves your device. When connecting to a backend, requests go only to the host you configure.
