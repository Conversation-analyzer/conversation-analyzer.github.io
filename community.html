Ways to Contribute

There are many ways to be part of the project, regardless of your experience level.

Report Bugs

Found something broken? Open an issue with steps to reproduce. The more detail you provide, the faster we can fix it.

Feature Requests

Have an idea for a new feature or connector? Share it in GitHub Discussions or file a feature request issue.

Discussions

Join conversations about architecture decisions, upcoming features, and how others are using the tool in their workflow.

Code Contributions

Pick up an issue labeled "good first issue" or "help wanted," fork the repo, and submit a pull request.

Documentation

Improve docs, fix typos, add usage examples, or write guides. Good documentation is just as valuable as good code.

Guidelines

Code of Conduct

We are committed to providing a welcoming and inclusive experience for everyone.

Be Respectful

Treat everyone with respect. We welcome contributors of all backgrounds and experience levels. Constructive feedback is encouraged; personal attacks are not.

Be Inclusive

We strive to create a positive environment. Use inclusive language, be mindful of different perspectives, and help newcomers feel welcome.

Be Professional

Keep discussions focused on the project. Disagreements are natural, but always engage in good faith and focus on the technical merits.

Report Issues

If you experience or witness unacceptable behavior, report it by emailing chnrndr3@email.com. All reports will be handled confidentially.
