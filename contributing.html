Contributing

We welcome contributions of all shapes and sizes. Whether you're fixing a typo, building a new connector, or improving the parser — here's how to get started.

Ways to Contribute

Everyone Can Help

You don't need to be an expert to make a meaningful impact. Here are many ways to contribute.

Report Bugs

Found something broken? Open an issue with clear reproduction steps. Include screenshots and environment details to help us fix it faster.

Suggest Features

Have an idea that would make the analyzer better? Share it in GitHub Discussions. We love hearing how you want to use the tool.

Build Connectors

Create connectors for new AI platforms. The abstract connector system makes it easy to add support for any chat export format.

Improve Parsers

Enhance the universal parser to handle edge cases, new formats, or richer metadata. Every improvement unlocks new insights.

Add Charts

Design new visualizations for token usage, latency, cost breakdowns, or agent comparison. Charts make data tell a story.

Improve Docs

Fix typos, write examples, add guides, or translate documentation. Great documentation is just as valuable as great code.

Setup

Development Setup

AI Conversation Analyzer has zero build dependencies. Here's everything you need to run it locally.

Prerequisites

  • A GitHub account
  • Git installed locally
  • A code editor (VS Code recommended)
  • A modern browser (Chrome, Firefox, Edge)
  • Basic familiarity with HTML, CSS, and JavaScript

Setup Commands

# Clone your fork
git clone https://github.com/YOUR_USERNAME/conversation-analyzer.github.io.git
cd conversation-analyzer.github.io

# Serve locally (any of these work)
python -m http.server 8000
# or
npx serve .
# or
php -S localhost:8000

PR Process

Pull Request Process

A clear, repeatable process keeps contributions smooth for everyone.

Step 1

Fork & Branch

Fork the repository and create a feature branch from main. Use a descriptive name like feat/csv-export or fix/tool-timeline.

Step 2

Write Code

Make your changes following the existing code style. Keep commits small and focused. Write clear commit messages in the present tense.

Step 3

Test Locally

Open the HTML files in your browser and verify everything works. Check console for errors. Test across at least one other browser if possible.

Step 4

Commit & Push

Commit your changes and push your branch to your fork. Follow the Conventional Commits format if possible.

Step 5

Open a Pull Request

Open a PR from your fork to the main repository. Describe what your changes do, why they're needed, and how to test them. Maintainers will review and provide feedback.

Guidelines

Contribution Guidelines

A few simple principles to keep the project healthy and welcoming.

Communicate First

Before starting significant work, open an issue or start a discussion. This avoids duplicate effort and ensures your contribution aligns with the project direction.

Keep It Focused

A single pull request should address one concern. If you have multiple unrelated changes, submit separate PRs. This makes reviews faster and more reliable.

Update Docs

If your changes introduce new behavior or modify existing functionality, update the relevant documentation. No contribution is complete without docs.

Be Respectful

All contributors must follow our Code of Conduct. Be constructive, patient, and welcoming. We're all here to learn and build something great together.
